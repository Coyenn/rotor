# rotor 2.0 — TOML config, `$asset` macro, bundler & QOL

**Date:** 2026-06-12 · **Status:** approved (user). Major version. Breaking config change.

2.0 is two headline changes plus a QOL/bundler sweep:

1. **Config moves from `rotor.config.ts` to `rotor.toml`** with a taplo `#:schema` directive (JSON Schema). The TS loader (goja/esbuild) survives only behind `rotor migrate`.
2. **`$asset("path/to/x.png")`** — a compile-time macro that returns a `rbxassetid://` string, backed by the lockfile cache, auto-uploading on a cache miss (when `ROBLOX_API_KEY` is set). Projects pick one asset delivery mode: generated `assets.luau` **or** the `$asset` transformer.

Everything else (compiler byte-parity, Luau toolchain, deploy) is unchanged and additive.

## Decisions (from user)

- **TOML replaces `.ts`.** `rotor.toml` is THE config. `rotor migrate` converts an existing `rotor.config.ts`. Schema via `#:schema ./rotor.schema.json` (taplo directive) + a rotor-generated `rotor.schema.json`.
- **`$asset` auto-uploads on cache miss** (needs `ROBLOX_API_KEY` at build time), persists the id to the lockfile, inlines it. Offline / no-key + cache miss → clear compile error pointing at `rotor asset sync`.
- **One asset mode per project**: `[assets] mode = "module" | "macro"`. Shared lockfile/cache.
- Plus: more QOL, bundling/importing features, robust auto type-companion generation (`rotor-env.d.ts`, `rotor-asset.d.ts`).

## Config — `rotor.toml`

```toml
#:schema ./rotor.schema.json

[assets]
mode = "macro"                  # "module" (assets.luau) | "macro" ($asset transformer)
paths = ["assets/**/*.png", "assets/**/*.ogg"]

[assets.output]                 # only used in "module" mode
luau = "src/shared/assets.luau"
types = "src/shared/assets.d.ts"

[assets.creator]
type = "group"                  # "user" | "group"
id = 12345

[deploy.environments.dev]
universeId = 111
[deploy.environments.dev.places.start]
file = "build/game.rbxl"
placeId = 222

[deploy.environments.prod]
universeId = 333
[deploy.environments.prod.places.start]
file = "build/game.rbxl"
placeId = 444
versionType = "published"
[deploy.environments.prod.experience]
name = "My Game"
playability = "public"
[deploy.environments.prod.gamepasses.vip]
name = "VIP"
price = 250
icon = "assets/vip.png"
[deploy.environments.prod.socials.discord]
title = "Join us"
url = "https://discord.gg/x"
type = "discord"
```

- **Loader** (`internal/config`): TOML via `github.com/BurntSushi/toml`. `Load(dir)` reads `rotor.toml`, returns `(*Config, error)` — same `Config` structs as 1.x (json+toml tags), so every consumer (`asset`, `deploy`, `doctor`) is untouched aside from the load call. `ErrNotFound` when absent. Validation unchanged (price ≥ 0, enums, etc.) plus `assets.mode ∈ {module, macro}`.
- **Schema** (`internal/config/schema.go`): a hand-maintained JSON Schema (draft-07) mirroring the structs 1:1 (the TOML analogue of the old `TypeDeclarations`). `RefreshSchema(dir)` writes `rotor.schema.json` atomically when missing/stale (the pattern `RefreshTypeDeclarations` used). `rotor init` writes it; `asset`/`deploy` refresh it on successful load. The config's first line is `#:schema ./rotor.schema.json` so taplo / Even Better TOML give validation + completion.
- **`rotor migrate`** (`cmd/rotor/migrate.go`): loads a legacy `rotor.config.ts` via the retained goja/esbuild path (now `loadLegacyTS`, the ONLY remaining goja user), serializes to `rotor.toml` (prepending the `#:schema` line) + writes `rotor.schema.json`, and renames the old file to `rotor.config.ts.bak`. Idempotent; errors clearly if no `.ts` present.
- `rotor.config.ts` / `rotor-config.d.ts` are dropped from `init`; goja/esbuild remain compiled in only for `migrate`.

## `$asset` macro (`internal/transformer` + `internal/compile` + `internal/assets`)

Parallel to `$env`:

- **Surface**: `$asset("assets/logo.png")` → inlined `"rbxassetid://123"` (a Luau string). Path is project-relative (or relative to the importing file if it starts with `./`).
- **Type surface**: synthetic ambient `declare function $asset(path: string): string;` injected into the program (same overlay mechanism as `$env`'s `__rotor_env.d.ts`). On-disk `rotor-asset.d.ts` auto-written by `build`/`check` when sources reference `$asset`, with the coexistence guard (skip synthetic injection when an identical on-disk decl is already in the program).
- **Resolver** plumbed via `State.Assets` (like `State.Env`): `Resolve(path) (id string, err error)`:
  1. in-memory cache (same asset referenced N times → one upload),
  2. `rotor-lock.json` keyed by content hash (cache hit → reuse id),
  3. cache miss + `ROBLOX_API_KEY` → upload via Open Cloud (`internal/cloud` CreateAsset + PollOperation, creator from `[assets.creator]`), record the new id,
  4. cache miss + no key → diagnostic `rotorAssetNotCached` ("asset not synced; run `rotor asset sync` or set ROBLOX_API_KEY").
- **Compile integration** (`internal/compile`): load the lockfile + creator once per build, construct the resolver (cloud client via `cloud.FromEnv`, may be nil), plumb to `State`; after a successful build, flush newly-uploaded entries back to `rotor-lock.json` atomically. The transformer never does file/network I/O directly — it calls the resolver, which records; the pipeline persists. Non-literal `$asset(x)` → clear diagnostic (no panic). Missing file → diagnostic.
- **Determinism**: a cache-hit build is fully offline/deterministic (parity-safe). Network only happens on a genuine miss with a key present.

### Asset delivery modes (`[assets] mode`)

- `"module"` (default, current behaviour): `rotor asset sync` uploads `paths` and writes `assets.luau` + `assets.d.ts`; `$asset` still works but the blessed path is the generated module.
- `"macro"`: `rotor asset sync` uploads `paths` and maintains the lockfile + writes `rotor-asset.d.ts` (no `assets.luau`); `$asset` is the consumption path, with build-time auto-upload filling any gaps.

Both modes share `internal/assets` scan/hash/upload/lockfile.

## Bundler & importing (`internal/bundle`)

Roadmap follow-ups, now shipped:

- **`.luaurc` alias resolution**: honor `aliases` in `.luaurc` so `require("@pkg/x")` resolves through the configured root during bundling.
- **Data-file embedding**: a `require("./data.json")` inlines the parsed JSON as a Luau table literal; `require("./x.txt")` inlines the file content as a string. Each becomes a cached module in the bundle (same run-once semantics as code modules).
- **`--exclude <glob>`** (repeatable): leave matching requires verbatim (not inlined) — for runtime-provided modules.

## QOL sweep (`cmd/rotor`)

- **`rotor clean [path]`**: remove the build outputs (`outDir`, `includePath`) and, with `--types`, the generated companions (`rotor-env.d.ts`, `rotor-asset.d.ts`). Prints what it removed; never touches source.
- **`--json` on `build` / `check`**: emit a machine-readable result object to stdout (`{version, files, durationMs, diagnostics:[{file,line,col,message,severity}]}`) instead of styled output, for CI/editor integration. Styled output stays the default.
- **`rotor add <pkg>...`**: add `@rbxts/*` (or any) dependency entries to `package.json` (loose `*`/`^` pins, like the init wizard), and remind to install. Pure package.json edit; no network.
- **Auto type-companion generation** (consolidated): one code path writes `rotor-env.d.ts` and `rotor-asset.d.ts` during `build`/`check` when their macros are referenced; `rotor.schema.json` refreshed by config-loading commands. All atomic, stale-checked, "generated by rotor; do not edit" headers, and ignored by the watcher.

## Testing

- `internal/config`: TOML golden fixtures (valid, invalid, mode enum), schema-content test (parses as JSON, contains every property), `migrate` round-trip (legacy .ts → toml → Load equals the .ts Load). No Node.
- `internal/transformer` + `internal/compile`: `$asset` end-to-end fixture — cache hit inlines the id (offline, with a pre-seeded lockfile), non-literal arg → diagnostic, missing file → diagnostic, miss+no-key → `rotorAssetNotCached`; a fake resolver covers the upload-on-miss path without network. Parity suites (`internal/diff`, `internal/conformance`) stay byte-green.
- `internal/bundle`: `.luaurc` alias resolution, JSON→table and txt→string embedding (bundle parses with `internal/luau/cst` and the data round-trips), `--exclude` leaves requires verbatim. Lune behavioral test still passes.
- `cmd/rotor`: `clean` removes only outputs; `--json` shape; `add` edits package.json; `migrate` writes toml+schema+bak.

## Non-goals (2.0)

- Hosting the JSON schema at a public URL (local relative `#:schema` is the default; a hosted URL is a later nicety).
- Dropping goja/esbuild entirely (kept for `migrate`; removable in a future major once migration is widespread).
- `rotor fmt` / the readable Luau pretty-printer (still a separate roadmap subproject).
- Cookie-auth asset types (animations) — Open Cloud only, unchanged.
